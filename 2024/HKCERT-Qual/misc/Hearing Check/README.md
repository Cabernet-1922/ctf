We notice that when the `Pitch_bend_c` follow with number 16383, then the first `note` right after this `Pitch_bend_c` will have a `+2` in the ascii number. For example, the first note that right above `Pitch_bend_c=16383` will have value `67+2=69`. Other than 16383, we have `{12288,8192,4096,0}`. \
`12288` will have `+1` in the ascii number, `8192` will remain unchanged, `4096` will have a `-1`, `0` will have a `-2`. With these rules:
```text
Pitch_bend_c, 0, 0 Perform -2
Note_on_c, 0, 67, 127 -> C-2=A

Pitch_bend_c, 0, 16383 Perform +2
Note_on_c, 0, 65, 119 A+2=C

Pitch_bend_c, 0, 16383 Perform +2
Note_on_c, 0, 67, 111 -> C+2=E

Pitch_bend_c, 0, 0 Perform -2
Note_on_c, 0, 72, 103 -> H-2=F

Pitch_bend_c, 0, 8192 Perform 0
Note_on_c, 0, 72, 96 -> H

Pitch_bend_c, 0, 0 Perform -2
Note_on_c, 0, 76, 88 -> L-2=J

Pitch_bend_c, 0, 16383 Perform +2
Note_on_c, 0, 74, 80 -> J+2=L

Pitch_bend_c, 0, 4096 Perform -1
Note_on_c, 0, 78, 72 -> N-1=M
```
